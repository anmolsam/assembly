// Capture layer · the real fork
Both modes. Same brain.

Every competitor picks one. We run both behind one intelligence layer, so you get fleet-wide coverage and invisible on-call coaching.

🤖

Recall.ai bot

server-side · centralized · visible

A bot joins the Meet from the connected calendar and streams real-time audio/transcript back. This is how Otter, Fireflies, Attention & Sybill capture.

  • Zero install — works across 100s of reps instantly
  • Clean per-speaker diarization out of the box
  • Survives a rep's laptop dying mid-call
  • Real-time WS and a final recording for the council pass
  • Trade-off: a visible "Notetaker" participant + makes recording explicit (often a compliance plus)
Best for fleet-wide coverage & compliance — the default for the rollout.
🫥

Desktop overlay

per-rep · local · invisible

A local app taps system audio and renders an always-on-top overlay only the rep sees — the Cluely approach (no bot ever joins).

  • No bot in the call — prospect never knows
  • Lowest latency (audio captured at the OS, straight to STT)
  • Works on any audio: Meet, Zoom, phone, in-person
  • Overlay hidden from screen-share*
  • Trade-off: per-rep install, machine-dependent, consent rules vary by region
Best for invisible, in-the-moment coaching — for reps who want the live whisper.

* Cluely markets its overlay as undetectable on screen-share; on modern macOS this is contested (ScreenCaptureKit can capture the composited display), and detector tools exist. We treat "hidden" as best-effort, not a guarantee.
